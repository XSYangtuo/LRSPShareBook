    调研结果省流版

    (使用GPT-4.5 获得的省流版，如果你不想看完整报告，可以阅读这20条，加粗是作者也深有同感的内容)

    1. 资料需求不是“期末限定”，而是平时就很高频，很多同学一学期会反复找 3 到 5 次甚至更多。
    2. 真正稀缺的不是资料总量，而是“快速搜索找到对的资料”的能力，这比“平台做得大不大”更重要。
    3. CC98 的优势主要在真题和经验，不在笔记；它更像信息集市，不像结构化资料库。
    4. GitHub 对“整理好的笔记”更有优势，但它的受众天然被技术门槛筛掉了一批。
    5. 平台之间不是简单替代关系，而是功能分工不同：CC98 找经验，GitHub 找体系化笔记，群聊负责临时分发。
    6. 问题不是“平台少”，而是入口太散：论坛、群文件、个人页、文档站都在存资料，但彼此不通。
    7. CC98 最大的问题不是没内容，而是内容太多又按时间堆着，越老越难找。
    8. 资源楼这种形式有天然上限：一页里同时塞回忆卷、笔记、经验，最后只能靠标记硬分，浏览体验会越来越差。
    9. 常见的“一个帖子打包一学期所有课程”对发布者方便，但对索引平台很不友好，因为链接没法精确跳到某门课。
    10. 搜索差几乎是跨平台共病，但在 CC98 上尤其严重；这不是用户不会搜，而是论坛结构本来就不是按课程建的。
    11. 手机端问题不是小毛病。问卷几乎全是手机上填的，说明学生本来就在移动端找信息，手机难用挺劝退的。（这点存疑？因为大家好像都用电脑办公）
    12. 群文件的问题不是“偶尔乱”，而是天然会失效、沉底、断链，所以适合临时救火，不适合长期沉淀。
    13. 商业文档平台的问题比“要不要付费”更复杂：审核、锁权、下架会让资源变成看得见但拿不到。
    14. 个人站最大的风险不是搭不起来，而是可信度和持续性都绑在一个人身上，人一忙，平台就容易半停更。
    15. 大二是资料需求最旺的时候，不是大四；说明平台最该服务的，其实是刚进入专业课压力区的人。
    16. 大一和大四都更依赖学长学姐，但原因不一样：大一是不知道去哪找，大四是找“经验型信息”比找资料本身更重要。
    17. GitHub 使用率会随年级上升，不是因为高年级更爱开源，而是他们终于跨过了 Git 和 Markdown 这道门槛。
    18. 85.6% 的人说愿意分享，说明供给端并不冷漠；真正的瓶颈是上传、排版、分类这些琐碎劳动。
    19. 上传里最重要的是时间成本；减少上传需要的时间和精力，大家不愿为分享再做一次作业。这一点CC98做的最好。
    20. 多份访谈都指向同一个硬结论：平台最难的不是建站，而是稿件输入、更新积压和接班传承。很多平台不是死于技术失败，而是死于热情耗尽和工作流过重。

    “最难的不是技术，是稿件输入和传承。” —— zjuer2080
